# Matrix Transpose 教学指南

## 这个练习在做什么

这个练习的目标是让你把一个二维矩阵从

- `A(i, j)` 复制到 `B(i, j)`，再进一步
- 把 `A(i, j)` 写到 `B(j, i)`

也就是先做 **Copy Kernel**，再做 **Transpose Kernel**。

这两个核函数很像，区别只在于：

- copy：输入和输出的线性索引一样
- transpose：输出的索引换了方向

---

## 你要理解的数据布局

代码里矩阵是一维数组存的，但逻辑上它是二维的。

例如：

- `sizeX` = 每一行有多少列
- `sizeY` = 一共有多少行

那么二维坐标 `(i, j)` 映射到一维数组的公式是：

```text
index = j * sizeX + i
```

这里：
- `i` 是 x 方向，也就是列
- `j` 是 y 方向，也就是行

所以矩阵按行优先存储。

---

## Copy Kernel 的思路

Copy Kernel 其实就是“每个线程搬一个元素”。

### 线程做什么

每个线程先算自己的全局坐标：

```text
i = blockIdx.x * blockDim.x + threadIdx.x
j = blockIdx.y * blockDim.y + threadIdx.y
```

然后：

1. 检查 `i` 和 `j` 有没有越界
2. 算出线性下标 `index = j * sizeX + i`
3. 执行 `b[index] = a[index]`

### 这段代码的意义

这一步的重点不是“拷贝”本身，而是：

- 你学会了二维线程如何对应二维数据
- 你学会了如何把二维坐标变成一维数组下标
- 你学会了如何防止越界

---

## Transpose Kernel 的思路

Transpose 和 Copy 的区别很小，但很关键。

### 输入索引

输入还是从 `A(i, j)` 读，所以输入索引不变：

```text
index_in = j * sizeX + i
```

### 输出索引

但是输出要写到转置位置，也就是 `B(j, i)`。

因为输出矩阵的宽高交换了，所以输出索引变成：

```text
index_out = i * sizeY + j
```

### 这一步的本质

把这一句记牢：

```text
A(i, j) -> B(j, i)
```

它意味着：
- 读的时候按原矩阵位置读
- 写的时候按转置后的位置写

---

## 为什么要先做 Copy Kernel

先做 Copy Kernel 是为了分步验证。

如果 Copy 都错了，那 Transpose 的问题就不好判断。

正确顺序是：

1. 先确认线程索引对不对
2. 再确认边界检查对不对
3. 再确认一维下标对不对
4. 最后再把输出索引改成转置版本

这样调试更容易。

---

## 线程块和网格怎么配

这个练习用的是二维线程块和二维网格。

### 线程块

```text
blockDim = (16, 16, 1)
```

意思是：
- 每个 block 有 16 x 16 = 256 个线程
- 每个线程负责一个矩阵元素

### 网格

```text
gridDim.x = ceil(sizeX / 16)
gridDim.y = ceil(sizeY / 16)
```

也就是：
- x 方向要多少个 block 才能覆盖所有列
- y 方向要多少个 block 才能覆盖所有行

### 为什么要向上取整

因为矩阵尺寸不一定正好是 16 的倍数。

比如：

- `sizeX = 1234`
- `sizeY = 3153`

就不能整除，所以要多开一些线程来覆盖最后的边缘。

多出来的线程靠边界检查挡住。

---

## 为什么要做边界检查

线程数通常会比数据量多一点。

例如：

- 矩阵只有 1234 列
- 但 block 是按 16 对齐开的
- 最后一组线程里会有一部分线程超出边界

所以每个线程都要检查：

```text
if (i >= sizeX || j >= sizeY) return;
```

这样就不会访问非法内存。

## 主函数里完整流程

你可以把 `main()` 理解成下面这几步：

### 1. 创建主机端数组

- `a`：输入矩阵
- `b`：输出矩阵
- `a_gold`：copy 的参考答案
- `b_gold`：transpose 的参考答案

### 2. 填充输入数据

把 `a` 填成递增数字，方便检查结果。

### 3. 计算 CPU 参考答案

CPU 端先算好正确结果，这样 GPU 跑完以后可以比较。

### 4. 分配 GPU 内存

- `d_a`
- `d_b`

### 5. 拷贝输入到 GPU

把 `a` 传到 `d_a`。

### 6. 启动 copy kernel

验证“线程索引 + 边界检查 + 线性下标”这套逻辑是否正确。

### 7. 启动 transpose kernel

把输出索引改成转置形式。

### 8. 把结果拷回 CPU

从 `d_b` 复制回 `b`。

### 9. 和 gold 比较

如果结果一致，说明实现正确。

### 10. 释放 GPU 内存

用 `cudaFree` 回收 `d_a` 和 `d_b`。

## 这个练习真正要学会的东西

### 1. 二维索引

你要能熟练写出：

```text
i = blockIdx.x * blockDim.x + threadIdx.x
j = blockIdx.y * blockDim.y + threadIdx.y
```

### 2. 行优先下标

你要能马上写出：

```text
index = j * width + i
```

### 3. 转置映射

你要知道：

```text
A(i, j) -> B(j, i)
```

### 4. 边界检查

你要知道多出来的线程不能乱写内存。

## 一句话总结

这个练习就是：

- 先学会用 CUDA 线程表示二维坐标
- 再把二维坐标映射到一维数组
- 最后把 copy 改成 transpose

只要你把“索引公式”和“边界检查”想清楚，代码本身就不难。
