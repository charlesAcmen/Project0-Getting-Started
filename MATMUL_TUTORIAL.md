# Matrix Multiplication 教学指南

## 这个练习在做什么

这个练习的目标是实现最基础的矩阵乘法：

```text
P = M × N
```

在代码里，每个 CUDA 线程负责算出 **P 的一个元素**。

这意味着：

- 一个线程 = 一个输出位置
- 一个线程要自己做一个小循环，把乘法和加法累积起来

---

## 先看你要理解的核心公式

矩阵乘法的标准形式是：

```text
P[row, col] = sum over k of M[row, k] * N[k, col]
```

也就是说：

- 从 M 的一整行取数据
- 从 N 的一整列取数据
- 做点积得到 P 的一个元素

---

## 代码里的索引约定

这个练习的代码采用的是二维线程索引：

```text
px = blockIdx.x * blockDim.x + threadIdx.x
py = blockIdx.y * blockDim.y + threadIdx.y
```

这里：
- `px` 表示输出矩阵的 x 坐标
- `py` 表示输出矩阵的 y 坐标

你可以把它理解成：

- `px` = 列号
- `py` = 行号

---

## kernel 里每个线程要做什么

### 1. 计算自己的输出坐标

每个线程先算出：

- `px`
- `py`

这样它就知道自己负责 P 的哪个元素。

### 2. 做边界检查

因为 launch 出来的线程数通常会比实际数据多一点，所以要判断：

```text
if (px >= sizeMX || py >= sizeNY) return;
```

如果越界就直接结束。

### 3. 计算点积

对每个 `k`：

```text
dot += M[...] * N[...]
```

这一步是矩阵乘法的核心。

### 4. 写回 P

最后把 `dot` 写进输出矩阵。

---

## 为什么 kernel 里会有一个 for 循环

因为一个输出元素不是简单复制来的，而是要把一整条乘积链累加起来。

例如，如果：

- M 的一行有 32 个元素
- N 的一列有 32 个元素

那么一个线程就要做 32 次乘法和加法。

所以：

- 外层并行：很多线程同时算很多个输出元素
- 内层串行：每个线程自己做点积循环

这就是 CUDA 里最常见的“**外并行、内串行**”模式。

---

## 主函数里的流程

### 1. 初始化尺寸

代码里现在先用了一个简单的正方形配置：

- `sizeMX = 32`
- `sizeXY = 32`
- `sizeNY = 32`

这样先容易验证。

后面你可以再尝试：
- 15 x 15 x 15
- 64 x 64 x 64
- 128 x 64 x 32
- 123 x 123 x 123

### 2. 分配主机内存

要准备四个数组：

- `matrixM`：输入矩阵 M
- `matrixN`：输入矩阵 N
- `matrixP`：GPU 算出来的结果
- `matrixPGold`：CPU 参考答案

### 3. 填充输入矩阵

现在代码里用随机数填充 M 和 N。

这样做的好处是：
- 不同位置的值不一样
- 更容易看出索引有没有写错

### 4. 在 CPU 上计算 gold

这一步是为了对比 GPU 结果。

如果 GPU 结果和 CPU 参考结果一致，说明你的 kernel 没问题。

### 5. 分配 GPU 内存

要给：

- `d_matrixM`
- `d_matrixN`
- `d_matrixP`

开辟显存。

### 6. 把输入拷到 GPU

把 M 和 N 送到 GPU 上。

### 7. 启动 kernel

把输出矩阵 P 的每个元素交给 GPU 线程并行计算。

### 8. 结果拷回 CPU

把 `d_matrixP` 拷回 `matrixP`。

### 9. 对比 gold

确认 GPU 和 CPU 的结果一致。

### 10. 释放 GPU 内存

把显存还回去。

## 线程块和网格怎么配

这个练习用了：

```text
blockDim = (16, 16, 1)
```

意思是一个 block 里有 256 个线程。

网格大小由输出矩阵决定：

```text
gridDim.x = ceil(sizeMX / 16)
gridDim.y = ceil(sizeNY / 16)
```

为什么是输出矩阵的尺寸？因为每个输出元素都要被算一次。

---

## 为什么要向上取整

因为尺寸经常不是 16 的倍数。

比如：

- 32 可以整除 16
- 123 不可以整除 16

所以必须多开一点线程，再靠边界检查过滤掉越界线程。

---

## 这道题最重要的地方

### 1. 你要能写出输出元素对应的线程坐标

```text
px = blockIdx.x * blockDim.x + threadIdx.x
py = blockIdx.y * blockDim.y + threadIdx.y
```

### 2. 你要能写出点积循环

```text
for k in [0, sizeXY)
    dot += M[...] * N[...]
```

### 3. 你要知道线程和循环的关系

- 一个线程算一个输出元素
- 一个输出元素内部还有一个小循环

### 4. 你要理解内存布局

这道题最容易出错的就是数组下标。

只要某个下标公式写错，结果就会整体不对。

- `16 x 16 x 16`
- `32 x 32 x 32`
- `64 x 64 x 64`
- `128 x 64 x 32`
- `123 x 123 x 123`
- `771 x 771 x 771`

如果你想练调试，可以故意：

- 改错一个索引公式

- 改错 grid 大小

然后看结果怎么坏掉。

---

## 一句话总结

Matrix multiplication 练的是：

- 2D 线程索引
- 边界检查
- 点积循环
- 正确的矩阵下标映射

如果你把这些理顺了，代码其实就是“每个线程算一个元素”。
